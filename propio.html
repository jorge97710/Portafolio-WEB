Si guieres ver el codigo de la mayoria de mis laboratorios, puedes verlo AQUI.




Un breve resumen de lo que se hizo en el curso y los temas que se vieron. Luego de esto se tiene informacion mas detallada acerca de cada laboratorio. Durante el curso de desarrollo y tecnologias WEB iniciamos todo viendo un repaso de como funcionan los protocolos en internet. Continuamos con el tema que nos llevo al laboratorio #1. Como funcionaba el internet antes de que existiera la WEB, con esto nos referimos a como funcionaba todo antes de que se tuvieran "navegadores", uno se conectaba por medio de comandos a lo que funcionaba como una PTY practicamente. Luego de esto inifiamos con los temas basicos del desarrollo en HTML . Con esto, viendo diferentes etiquetas o tags, el uso de los mismos. La estructura correcta de un documento HTML y terminando por realizar nosotros una breve historia como lo puede ser el laboratorio #2.

Como siguiente tema empezamos a ver acerca de lo "estetico" en la WEB. Como darle diferentes estilos a nuestras paginas. Logrando esto por medio de CSS. Aprendimos a darle estilo y estetica a nuestras paginas para que no sean "texto plano". Con esto pudimos ver como tambien se puede crear diferentes objetos utilizando solo CSS como se puede ver en el laboratorio #3.

Luego de ver HTML y CSS empezamos a ver que para darle funcionalidad a una pagina, era necesario el uso de JavaScript. Aprendiendo asi diferentes funciones de las que existen en este. Dentro de las funciones realizables dentro de JavaScript, aprendimos a comunicarnos con un servidor (tanto para recibir como para enviar datos). Esto lo logramos mediante la instruccion Fetch. Con esto pudimos crear un chat como en el laboratorio #4

Despues de haber visto HTML, CSS, JS cambiamos un poco el rumbo. Con esto me refiero a que empezamos a ver un entorno en tiempo de ejecucion. Siendo este Node JS. Con este entorno y ciertas bibliotecas, nos propusimos a hacer el laboratorio #5 que es practicamente como el cuarto, sin embargo en este la carga no la maneja "la pagina"

Como siguiente punto iniciamos a ver React, webpack, babel y tecnologias similares. Todas estas para una interfaz de usuario. Con esta creamos el laboratorio #6 que es un juego de memoria

Continuamos con el laboratorio #7 que es la implementacion y union por asi decirlo del laboratorio 4 o 5y el 6. Esto ya que en el mismo se utilizan datos de un servidor, y se interpretan apra crear una pagina con react. Esta sera un juego de laberinto

Y por ultimo tenemos el tema de linting y tests. Aunque en anteriores pasos habiamos platicado de este tema, este es un laboratorio en el que cobra relevancia. el laboratorio #8 consta de la implementacion de una calculadora, en la cual una de las ideas es hacer tests de funcionamiento

Laboratorio #4

El laboratorio #4 consta en una pantalla que funciona como medio de comunicacion con un servidor. En esta caso nuestra pantalla sera un cliente. Laboratorio #5

Laboratorio #5

El laboratorio #5 en principio tiene la misma funcion que el laboratorio 4. Este es un chat, con la diferencia de que ahora nuestra pantalla de chat no es la que se encarga de manejar todo el procesamiento de los mensajes. Esta funcion se realiza por parte de un servidor que nosotros desarrollamos, este servidor se conecta a nuestra pantalla de chat que funciona otra vez como un cliente. El servidor se encarga de descargar los mensajes, y si hay mensajes nuevos cada vez que los haya, se recarga la pantalla de mensajes en nuestro cliente, por otro lado, cada vez que uno desea mandar un mensaje, este se manda de nuestra pantalla de chat al servidor de node, y este lo parsea y lo postea en el servidor del chat general

si deseas acceder al codigo fuente entra aca: Laboratorio #5 Node


Laboratorio #6

Laboratorio #6

El laboratorio #6 consta de un grid de cuatro por cuateo. dentro de este grid hay un juego de memoria en el cual cuando se voltean dos tarjetas, se verifica si son iguales y si lo son se quedan boca arriba, y de lo contrario regresan a la orientacion original. Cuando todas las tarjetas se han volteado podemos decir que se gano el juego. Esta memoria fue implementada en react, creando los diferentes componentes y logica de los que esta formado el juego en si. A su vez se preparo todo el entorno como si este fuera un boilerplate, para poder desarrollar en el mejor ambiente posible la aplicacion en cuestion. Para esto se tuvo la implementacion de webpack con una serie de loaders para encargarse de la generacion del HTML, el manejo del SASS (scss), el uso de javascript, para el manejo de imagenes. Todos estos se encuentran propiamente configurados en webpack

si deseas acceder al codigo fuente entra aca: Laboratorio #6 ReactMemory


Laboratorio #7

Laboratorio #7

El laboratorio #7 para fines practicos es la uion de dos de las tecnolocias que hemos utilizado en el pasado. Por un lado para dicho laboratorio utilizaremos fetch, para comunicarnos con el servidor. Esto con el proposito de conseguir informacion por parte del servidor, informacion que sera utilizada para la creacion de un juego de laberinto. Ahora para mencionar la segunda tecnologia que utilizaremos en el desarrollo de este proyecto, tenemos REACT. Con react nos encargaremos de la creacion de los componentes que conformaran el juego. Estos se manejaran con dicha biblioteca, y la informacion para la creacion de los mismos se obtiene con fetch. A la larga este laboratorio es como la mezcla entre el Laboratorio #4 o #5 y el Laboratorio #6. Uniendo dos partes de nuestros conocimientos y habilidades nuevas

Laboratorio #8

Laboratorio #8

El laboratorio #8 consta de una calculadora. Para esta no era requisito el utilizar react ni tecnologias Similares. Esta calculadora cuenta con todas las funciones que tiene una normal, Estas son, suma,multiplicacion,division,resta. Teniendo en consideracion tambien el uso de numeros con punto decimal, y que las restas no puedan ser negativas. A su vez se tiene de extra la operacion "modulo". Como dato adicional cabe mencionar que en esta calculadora el display es de 9 digitos, por lo tanto no se pueden ingresar mas, ni se deben regresar mas al realizar las debidas operaciones.
